## Prerequisites

Before you begin, ensure you have:

- **Node.js 22.x** or higher installed
- A **Stripe account** ([sign up for free](https://stripe.com))
- Your Stripe API keys (test mode recommended for development)
- A code editor and terminal

## Installation

<Steps>
  <Step title="Clone or download the project">
    If you haven't already, get the project files and navigate to the directory:
    
    ```bash
    cd path/to/api
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using your preferred package manager:
    
    <CodeGroup>
    ```bash npm
    npm install
    ```
    
    ```bash yarn
    yarn install
    ```
    
    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>
    
    This installs:
    - Express server framework
    - Stripe Node.js SDK (v11.6.0)
    - CORS and body parsing middleware
    - Swagger for API documentation
    - Development tools (nodemon, jest)
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the root directory by copying the example:
    
    ```bash
    cp .env.example .env
    ```
    
    Open `.env` and add your Stripe credentials:
    
    ```bash .env
    # Stripe secret key (required)
    STRIPE_SECRET_KEY=sk_test_your_actual_secret_key
    
    # Stripe webhook signing secret (required for webhooks)
    STRIPE_WEBHOOK_SECRET=whsec_your_actual_webhook_secret
    
    # API configuration
    PORT=3000
    PUBLIC_API_URL=http://localhost:3000
    NODE_ENV=development
    ```
    
    <Note>
      Get your test API keys from the [Stripe Dashboard](https://dashboard.stripe.com/test/apikeys). Use test keys (starting with `sk_test_`) for development.
    </Note>
    
    <Warning>
      Never commit your `.env` file to version control. The `.env.example` file is provided as a template only.
    </Warning>
  </Step>

  <Step title="Start the development server">
    Launch the API server with hot reloading:
    
    <CodeGroup>
    ```bash npm
    npm run dev
    ```
    
    ```bash yarn
    yarn dev
    ```
    
    ```bash pnpm
    pnpm dev
    ```
    </CodeGroup>
    
    You should see:
    ```
    🚀 Server running on port 3000
    ```
    
    The API is now available at `http://localhost:3000`
  </Step>

  <Step title="Verify installation">
    Open your browser and navigate to the Swagger documentation:
    
    ```
    http://localhost:3000/doc
    ```
    
    You should see the interactive API documentation with all available endpoints.
  </Step>
</Steps>

## Make Your First API Call

Let's create a customer in your Stripe account:

<Steps>
  <Step title="Create a customer">
    Use curl to create a new customer:
    
    ```bash
    curl -X POST http://localhost:3000/api/customers \
      -H "Content-Type: application/json" \
      -d '{
        "email": "customer@example.com",
        "name": "John Doe",
        "phone": "+1234567890"
      }'
    ```
    
    Or use Postman/Insomnia with this request body:
    
    ```json
    {
      "email": "customer@example.com",
      "name": "John Doe",
      "phone": "+1234567890"
    }
    ```
  </Step>

  <Step title="Review the response">
    You should receive a successful response:
    
    ```json
    {
      "status": true,
      "statusCode": 201,
      "message": "Cliente creado correctamente",
      "data": {
        "id": "cus_xxxxxxxxxxxxx",
        "email": "customer@example.com",
        "name": "John Doe",
        "phone": "+1234567890"
      }
    }
    ```
    
    Save the `id` field - you'll need it for creating payments.
  </Step>

  <Step title="Verify in Stripe Dashboard">
    Log into your [Stripe Dashboard](https://dashboard.stripe.com/test/customers) and navigate to Customers. You should see your newly created customer.
  </Step>
</Steps>

## Create a Payment

Now let's create a payment intent for the customer:

```bash
curl -X POST http://localhost:3000/api/payments \
  -H "Content-Type: application/json" \
  -d '{
    "customer_id": "cus_xxxxxxxxxxxxx",
    "amount": 50,
    "payment_method": "pm_card_visa"
  }'
```

<Note>
  The `amount` is in your currency's base unit (e.g., dollars/euros). The API automatically converts it to cents (multiplies by 100) before sending to Stripe.
</Note>

Expected response:

```json
{
  "status": true,
  "statusCode": 201,
  "message": "Intento de pago creado correctamente",
  "data": {
    "id": "pi_xxxxxxxxxxxxx",
    "amount": 5000,
    "currency": "eur",
    "status": "requires_confirmation",
    "customer": "cus_xxxxxxxxxxxxx"
    // ... additional Stripe payment intent fields
  }
}
```

## Testing with Stripe Test Cards

Stripe provides test card numbers for different scenarios:

| Card Number | Scenario |
|------------|----------|
| `4242424242424242` | Successful payment |
| `4000000000000002` | Card declined |
| `4000002500003155` | Requires authentication (3D Secure) |

Use any future expiry date and any 3-digit CVC for testing.

For more test cards, see the [Stripe Testing Documentation](https://stripe.com/docs/testing).
